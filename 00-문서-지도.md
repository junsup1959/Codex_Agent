# 오케스트레이션 하네스 문서 지도

> [!important]
> 이 폴더는 개인 학습 및 운영 기록이다. 실제 런타임 원본은 `${CODEX_HOME}` 아래의 `AGENTS.md`, `agent-architecture`, `skills`, `agents`, validator 스크립트다. Obsidian 문서는 구조를 설명하고 기억하기 위한 해설서이며, validator의 source of truth가 아니다.

## 현재 구조 한 줄 요약

현재 Codex Agents 구조는 “모든 단계를 subagent로 소환하는 구조”가 아니다. 컨트롤 단계는 main agent가 필수 skill을 활성화해 role pass로 수행하고, 실제 물리 child agent는 validated `launch_manifest`에서 나온 worker/reviewer specialist lane에만 제한적으로 생성된다. run 단위 context는 물리 `context-manager`가 아니라 `codex-context-ledger` MCP가 맡는다.

## 핵심 용어

| 용어 | 의미 |
| --- | --- |
| `architecture_required=true` | 단순 답변이 아니라 전역 오케스트레이션 구조를 따라야 하는 작업이라는 플래그 |
| main-agent role pass | 별도 child agent를 만들지 않고 현재 main agent가 특정 stage owner 역할을 수행하는 방식 |
| physical child agent | `spawn_agent`로 실제 생성되는 worker/reviewer specialist |
| `launch_manifest` | router가 만든 논리 실행 계획. 아직 실제 child id나 wait handle은 없다. |
| `active_passes` | distributor가 물리 child 생성 후 기록하는 실제 실행 원장 |
| `handoff_result` | worker/reviewer child가 반환하는 결과 packet |
| `aggregation_packet` | aggregator가 worker 결과를 정규화해 meta/review 단계로 넘기는 packet |
| `judgment_envelope` | meta-judge/feedback-synthesizer가 final 또는 feedback을 결정하는 envelope |
| `feedback_gate_evidence` | final claim을 허용하기 위한 필수 gate evidence |
| `contract_provenance` | stage artifact가 어떤 문서와 contract에서 나왔는지 가리키는 출처 정보 |

## 현재 운영 원칙

1. `AGENTS.md`는 compact pointer다. 긴 운영 계약은 `agent-architecture/*.md`로 보낸다.
2. runtime 절차는 `09-runtime-orchestration-steps.md`가 기준이다.
3. 필수 stage skill은 main agent가 직접 활성화한다.
4. 각 필수 skill은 `SKILL.md`와 `contract.json`을 함께 가진다.
5. hook 수준의 빠른 drift 검사는 `validate-skill-contracts.py`가 맡는다.
6. full architecture 승인은 `validate-agent-architecture.py`가 맡는다.
7. 실제 stage artifact는 `validate-runtime-artifact.py`, `harness_handoff.py`, `harness_gate.py`를 통과해야 한다.
8. final claim은 `meta_judge_stage_pass_ref`, reviewer evidence 또는 waiver, `feedback_gate_evidence` 없이는 금지된다.

## Canonical Loop

```text
Orchestrator <-> Context Manager -> Task Planner -> Worker Router
-> Specialist Worker Layer -> Aggregator -> Review Router
-> Specialist Review Layer -> Meta Judge -> Feedback Trigger Gate
-> Feedback to Orchestrator then Context Manager restart or Final Output
```

현재 구현에서는 `Context Manager`가 물리 resident agent가 아니다. main agent가 context-manager role pass를 수행하면서 `codex-context-ledger` MCP를 읽고 쓴다.
