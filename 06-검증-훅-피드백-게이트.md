# 검증, 훅, 피드백 게이트

> [!summary]
> 현재 구조는 prompt-only 규칙이 아니다. Python validator, runtime gate, smoke fixture, session replay, skill contract 검사가 함께 작동한다.

## 검증 계층

| 계층 | 스크립트 | 역할 |
| --- | --- | --- |
| skill contract preflight | `validate-skill-contracts.py` | hook에서 빠르게 돌릴 수 있는 `contract.json` drift 검사 |
| static architecture inventory | `validate-agent-architecture.py` | 문서, prompt, skill contract, fixture 연결 검증 |
| runtime artifact shape | `validate-runtime-artifact.py` | stage output JSON shape와 branch 검증 |
| handoff wrapping | `harness_handoff.py` | stage output을 next owner input으로 포장 |
| hard gate | `harness_gate.py` | invalid/stale/missing evidence면 다음 단계 차단 |
| synthetic smoke | `validate-runtime-gate-smoke.py` | positive/negative fixture로 runtime gate 회귀 검증 |
| physical replay | `validate-session-runtime.py` | 실제 session JSONL에서 spawn/wait/final gate 위반 탐지 |
| skill metadata | `quick_validate.py` | skill package와 metadata 검증 |

## hook의 역할

hook은 정책을 해석하는 곳이 아니라 빠른 검증 실행기다.

현재 적합한 hook 대상:

```powershell
python C:\Users\junsu\.codex\agent-architecture\validate-skill-contracts.py
```

이 스크립트는 필수 stage skill과 optional `$docker-memory`의 `contract.json`을 확인한다. full validator보다 가볍고, skill/contract drift를 빠르게 잡는 용도다.

hook이 대체하지 않는 것:

- full architecture validation
- runtime artifact validation
- reviewer evidence
- meta-judge
- feedback gate

## full validator

최종 승인 전에는 다음을 실행한다.

```powershell
python C:\Users\junsu\.codex\agent-architecture\validate-agent-architecture.py
```

주요 검사:

- required architecture files 존재
- `AGENTS.md` line limit
- canonical loop consistency
- deprecated contract token 금지
- recursive TOML catalog parse
- category routing contract
- launch manifest prompt gate
- meta prompt chain contract
- stage skill contract check
- routing collection fixture
- feedback loop gate fixture
- runtime smoke 실행

## runtime gate

각 stage artifact는 다음 경로를 거쳐야 한다.

```text
stage output
-> validate-runtime-artifact.py
-> harness_handoff.py
-> harness_gate.py
-> next owner
```

이 과정에서 다음을 확인한다.

- artifact branch가 stage owner와 맞는지
- required field가 있는지
- `context_packet_version`이 맞는지
- `contract_provenance`가 있는지
- lane이 materialized/waited/classified 되었는지
- reviewer evidence 또는 waiver가 있는지
- MCP cleanup snapshot이 clean인지

## Feedback Gate 필수 조건

`feedback_gate_mandatory=true`는 architecture-required run에서 항상 적용된다.

final, ready, approve, merge-ready, completion claim 전에 필요한 것:

- current `judgment_envelope`
- `meta_judge_stage_pass_ref`
- reviewer `review_input_refs` 또는 explicit `review_waivers`
- non-empty `feedback_gate_evidence`
- 최신 artifact와 gate evidence의 freshness

## Fail-Closed 조건

다음이면 final output 금지다.

- `feedback_required=true`
- `feedback_gate_evidence` 누락
- reviewer evidence 누락 및 waiver 없음
- materialized child가 wait되지 않음
- spawn failure가 unclassified
- timeout lane이 unclassified
- schema-invalid output을 정상으로 취급
- stale `context_revision`
- missing `contract_provenance`
- open MCP process 존재
- `mcp_quiescence_snapshot.cleanup_status != clean`
- previous judgment가 새 artifact를 승인하는 데 재사용됨

fail-closed는 “안전하게 멈춘다”가 아니라 “완료 claim을 금지하고 bounded feedback 또는 repair route로 보낸다”는 의미다.

## session replay validator

`validate-session-runtime.py`는 실제 Codex session JSONL을 재생해 다음 문제를 잡는다.

- `fork_context=true`와 explicit `agent_type`을 함께 사용
- router manifest 없이 specialist 직접 spawn
- failed spawn을 성공처럼 aggregation
- materialized child가 `wait_agent` 대상이 아님
- `close_agent`만 있고 wait evidence 없음
- prose-shaped aggregator
- final meta-judge prompt에 `stage_passes`, `active_passes`, reviewer evidence 없음
- open tool calls 또는 MCP residue가 남은 상태에서 final claim
